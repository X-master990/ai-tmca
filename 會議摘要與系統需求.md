## 一、會議概述

這兩段是**同一場工作會議**的錄音(中間有用餐),由公司人員與**系統開發者**討論一套「**音樂授權/著作權申請管理系統**」的需求與修改方向。會議中夾雜大量與工作無關的私人閒聊(本文件已濾除)。

核心議題:
1. 申請表單欄位的精簡與重排
2. 續約流程自動化(名單、續約函、信封、發票)
3. 總表(線上共用資料庫)的編輯與安全機制
4. 店家 ID 編碼標準化
5. 授權期間/日期判斷的 bug 修正

## 二、重點摘要

### 檔案 1 `2.m4a`(約 24 分)— 申請表單與欄位整理

- **計量方式**:總表統一用「數量」;單位分「台」(電腦伴唱機)與「場」(場次)。
- **單場式類別**有獨立欄位:歌曲、演出曲目、演出地點、節目名稱。需可在「總表」與「單場式表」之間切換檢視,並避免同一筆資料重複輸入。
- **欄位精簡**:刪除「性質」(已分類不需要)、信紙;調整「統編 / 發票抬頭 / 二聯 / 桶編」位置。
- **聯絡人欄位**:區分「申請人」與「現場聯絡人」(選填);電話拆為室內/行動/分機/傳真;刪除身份證資料(個資考量)。
- **寄送地址**:增加可勾選寄到店家或指定地址。
- **改動範圍**:優先處理「電腦伴唱機(搬上機)、營利性」,其他類別量少先不動。

### 檔案 2 `吉品初筵.m4a`(約 54 分)— 系統流程走查 + 續約/發票

- **搜尋**:支援全類別搜尋與單類別搜尋。
- **續約流程**:篩出未續約名單 → 自動生成續約行 → 列印續約函/證書 + 信封。
- **新增**:以新申請書新增;貼上店名自動帶出舊資料。
- **總表**:線上即時共用,承辦/會計/法務皆可檢視與編輯;需防誤改機制。匯入資料約 10% 有錯誤,須逐筆線上修正。
- **ID 編碼**:固定 7 碼(配合銀行系統),一店家一 ID,依「持證者」分類。
- **授權期間**:自動計算(到期日次日起算);目前日期判斷有 bug 待修。
- **完整續約示範**:新增資料 → 送出 → 進總表 → 會計開發票 → 發票號碼/日期自動帶入 → 匯出 Excel。
- **發票**:二聯/三聯/作廢可下拉(自動帶+可手動改);金額由會計輸入。

## 三、系統新增/修改功能清單(給開發者)

### A. 申請表單(欄位整理)

| # | 功能 | 說明 | 優先 |
|---|------|------|------|
| A1 | 刪除「性質」欄位 | 已改用分類,此欄不再使用 | 高 |
| A2 | 刪除「信紙」欄位 | 不需要 | 中 |
| A3 | 刪除身份證欄位 | 個資考量,改為「現場人員」 | 高 |
| A4 | 聯絡人拆分 | 區分「申請人」與「現場聯絡人」(選填) | 高 |
| A5 | 電話欄位拆分 | 室內電話 / 行動電話 / 分機 / 傳真,各自獨立欄位 | 高 |
| A6 | 統編/發票抬頭/二聯/桶編重排 | 桶編改為後端註記、不顯示於主畫面;同編號合併欄位 | 中 |
| A7 | 寄送地址選項 | 可勾選「寄店家」或「寄指定地址」 | 中 |
| A8 | 計量單位 | 總表統一用「數量」,值為「台」或「場」 | 中 |

### B. 單場式 / 檢視切換

| # | 功能 | 說明 | 優先 |
|---|------|------|------|
| B1 | 單場式專屬欄位 | 歌曲、演出曲目、演出地點、節目名稱 | 高 |
| B2 | 檢視切換 | 可在「總表(全欄位)」與「作業檢視(精簡欄位)」間切換 | 高 |
| B3 | 避免重複輸入 | 節目名稱等資料只輸入一次,自動帶入相關位置 | 高 |

### C. 續約流程

| # | 功能 | 說明 | 優先 |
|---|------|------|------|
| C1 | 未續約名單篩選 | 依到期日篩出當月未續約店家,可分類別 | 高 |
| C2 | 一鍵生成續約行 | 對未續約資料點擊即生成續約行 | 高 |
| C3 | 續約函列印 | 合併列印續約函/證書,模板依類別不同 | 高 |
| C4 | 信封列印 | 共用單一信封模板 | 中 |
| C5 | 辦理項目自動帶值 | 續約時自動填「續約」 | 中 |

### D. 總表(共用資料庫)

| # | 功能 | 說明 | 優先 |
|---|------|------|------|
| D1 | 線上即時共用 | 承辦/會計/法務同步看到變更 | 高 |
| D2 | 全欄位可編輯 | 指定人員可全編 | 高 |
| D3 | 防誤改機制 | 還原鍵 + 修改需「確認變更」對話框 | 高 |
| D4 | 逐筆修正匯入錯誤 | 匯入資料約 10% 有誤,支援線上逐筆修正 | 中 |
| D5 | 匯出/截圖列印 | 總表可匯出 Excel 或列印完整畫面 | 中 |
| D6 | 搜尋 | 全類別搜尋 + 單類別搜尋(人名、發票號碼…) | 高 |
| D7 | 年份篩選 | 可依年份檢視/篩選 | 中 |

### E. 店家 ID 編碼

| # | 功能 | 說明 | 優先 |
|---|------|------|------|
| E1 | 固定 7 碼 ID | 配合銀行系統,長度固定 7 碼 | 高 |
| E2 | 一店家一 ID | 同一持證者(總公司)共用同一 ID | 高 |
| E3 | 依持證者分類 | 以「持證者」而非「發票抬頭」做分類(連鎖店分店不同) | 高 |
| E4 | 類別前綴 | 首碼/前綴代表類別,建議用數字 | 中 |

> **類別清單**:電腦伴唱機(營利性 + 管委會,同價)、公益性、文化教育類、街頭藝人、評數、社區。

### F. 授權期間 / 日期(Bug 修正)

| # | 功能 | 說明 | 優先 |
|---|------|------|------|
| F1 | 到期日自動計算 | 新授權期間 = 舊到期日次日起算 | 高 |
| F2 | 日期判斷 bug | 修正被切到 12/31、民國 115/116(2026/2027)年份混亂的問題 | 高 |
| F3 | 申請日期自動帶值 | 自動帶電腦日期,或改為匯款日期手動輸入 | 中 |

### G. 發票(會計端)

| # | 功能 | 說明 | 優先 |
|---|------|------|------|
| G1 | 開立發票需有金額 | 金額由會計輸入,有金額才能開立 | 高 |
| G2 | 發票號碼/日期自動帶入 | 開立後自動寫回,並匯入 Excel | 高 |
| G3 | 二聯/三聯/作廢下拉 | 可自動帶值,並允許手動更改 | 高 |
| G4 | 會計專屬按鈕 | 提供會計列印發票用的入口 | 中 |

## 四、待釐清事項

- ID 是否需要英文前綴?(銀行端無英文代碼,可能需改用純數字分類)
- 管委會是否併入電腦伴唱機類別?(價格相同但表格不同,待確認)
- 傳真欄位是否保留?(暫留,確定無人使用再刪)
- 申請日期採「電腦日期」或「匯款日期」?(二擇一)
