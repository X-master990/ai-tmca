# ADR-002：TMCA 系統 — 技術選型與分階段實作藍圖

**狀態：** Accepted（接續 ADR-001 選 Option B）
**日期：** 2026-05-16
**承接：** ADR-001 §四 Option B

---

## 一、決策摘要

承 ADR-001 採 **Option B：自製 Web App**，本 ADR 鎖定具體技術選型與 8 階段交付藍圖。

| 層級 | 技術 | 理由 |
|---|---|---|
| 前端框架 | **React 18 + Vite + TypeScript + TailwindCSS** | 主流、學習曲線平緩、開發體驗最快 |
| 試算表 UI | **Univer** (`@univerjs/core`) | 開源、最像 Excel、繁體中文社群活躍、支援公式與多 sheet |
| 後端框架 | **Python 3.11 + FastAPI** | 自動 OpenAPI 文件、與 pandas/python-docx 同生態 |
| ORM | **SQLAlchemy 2.0 + Alembic** | 業界標準、遷移腳本好管 |
| 資料庫 | **PostgreSQL 15**（prod）/ **SQLite**（dev） | 12 個分頁的 JOIN/全文檢索皆穩 |
| Word 套印 | **docxtpl**（基於 jinja2 + python-docx） | 沿用 TMCA 既有 27 個 `.docx` 模板，幾乎零改寫 |
| Excel 匯入/匯出 | **openpyxl + pandas** | 一次性把 `110-115年 總表.xlsx` 灌進 DB |
| 認證 | **JWT + bcrypt + Cookie HttpOnly** | 4 個帳號規模，簡單夠用 |
| 部署 | **Docker Compose + Caddy（自動 HTTPS）+ DigitalOcean Droplet $12/月 + Managed Postgres $15/月** | 月費可控、單一指令上線 |
| CI | **GitHub Actions**（後續可加） | 自動跑 test、build image |

**為什麼前端選 Univer 而非 Handsontable / AG-Grid：**
- Univer 是「真正的試算表體驗」，公式列、儲存格樣式、多工作表分頁全部 Excel-like。
- 商業授權上 Univer Apache 2.0 完全免費，協會用無顧慮。
- Handsontable 商業版要錢、AG-Grid Enterprise 也要錢。

## 三、資料模型（DB Schema 概要）

### 主表：`records`

```sql
CREATE TABLE records (
  id              SERIAL PRIMARY KEY,
  category        VARCHAR(40) NOT NULL,   -- 12 種申請類型
  cert_no         VARCHAR(40) UNIQUE,     -- 證書編號
  issued_date     DATE,                   -- 發證日
  note            TEXT,                   -- 註記

  -- 發票
  invoice_date    DATE,
  invoice_type    VARCHAR(20),
  invoice_title   VARCHAR(200),
  tax_id          VARCHAR(20),
  invoice_no      VARCHAR(40),
  amount          INTEGER,

  -- 承辦
  source          VARCHAR(20),            -- 提報：承辦 / 自辦
  officer         VARCHAR(40),            -- 承辦人
  action_type     VARCHAR(20),            -- 辦理項目：新申辦 / 續約 / 授權延長
  apply_date      DATE,

  -- 申請人
  applicant_name  VARCHAR(80),
  applicant_id    VARCHAR(20),            -- 身分證
  applicant_mobile VARCHAR(30),
  applicant_phone  VARCHAR(30),
  applicant_fax    VARCHAR(30),

  -- 持證者（續約比對關鍵）
  holder_name     VARCHAR(120),
  holder_type     VARCHAR(40),

  -- 使用地址
  use_zip         VARCHAR(10),
  use_address     TEXT,

  -- 現場
  onsite_name     VARCHAR(40),
  onsite_mobile   VARCHAR(30),
  onsite_phone    VARCHAR(30),
  onsite_ext      VARCHAR(20),
  onsite_fax      VARCHAR(30),

  -- 標的
  qty             INTEGER,
  brand           VARCHAR(80),
  serial_no       VARCHAR(120),

  -- 授權期間（續約比對關鍵）
  period_start    DATE,
  period_end      DATE,

  -- 寄證地址（新增「確認」欄位）
  mail_type       VARCHAR(10),            -- 新欄位：掛號 / 平信
  mail_zip        VARCHAR(10),
  mail_address    TEXT,
  mail_recipient  VARCHAR(80),
  mail_phone      VARCHAR(30),

  -- 系統狀態欄位（新）
  issuance_status VARCHAR(10) DEFAULT '紅', -- 紅 / 綠
  renewal_status  VARCHAR(10),              -- 紅 / 綠 / null（無關續約）

  -- 稽核
  created_by      INTEGER REFERENCES users(id),
  created_at      TIMESTAMP DEFAULT NOW(),
  updated_by      INTEGER REFERENCES users(id),
  updated_at      TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_records_category    ON records(category);
CREATE INDEX idx_records_holder_taxid ON records(holder_name, tax_id);   -- 續約比對
CREATE INDEX idx_records_period_end  ON records(period_end);             -- 續約比對
CREATE INDEX idx_records_invoice_no  ON records(invoice_no);             -- 核發狀態查詢
```

### 角色與權限：`users`、`roles`

```sql
CREATE TABLE users (
  id            SERIAL PRIMARY KEY,
  username      VARCHAR(40) UNIQUE NOT NULL,
  password_hash VARCHAR(120) NOT NULL,
  display_name  VARCHAR(40),
  role          VARCHAR(20) NOT NULL,  -- officer_a / officer_b / accountant / issuer / viewer
  is_active     BOOLEAN DEFAULT TRUE,
  created_at    TIMESTAMP DEFAULT NOW()
);
```

| role | 可讀 | 可寫 |
|---|---|---|
| `officer_a` | 全部 | 全部（負責 `category = '單場次表演'`） |
| `officer_b` | 全部 | 全部（其餘 11 個 `category`） |
| `accountant` | 全部 | 發票欄位 + 自己工作清單 |
| `issuer` | 全部 | 核發欄位（發票號碼補登、產生按鈕） |
| `viewer` | 全部 | 唯讀 |

註：權限「寫」走 row-level 控制 + 欄位白名單。

### 其他表

- `templates`：模板 metadata（檔名、適用 category、type=證書/信封/通知函）
- `generated_files`：產出記錄
- `audit_log`：欄位修改前後值、誰、何時

## 四、8 階段交付藍圖

### Phase 0：專案骨架（本次對話可完成）
**交付**：
- `/Users/b/Documents/Claude/Projects/TMCA/app/` 完整資料夾結構
- `backend/`：FastAPI 起點、`requirements.txt`、SQLAlchemy 模型、Alembic 初始 migration
- `frontend/`：Vite + React + TS + Tailwind 起點、空白頁面
- `docker-compose.yml`、`Caddyfile`、`.env.example`
- `README.md` 寫清楚怎麼跑

**驗收**：本地 `docker compose up` 跑得起來，瀏覽器看到「TMCA 系統 - 開發中」首頁。

### Phase 1：認證 + 角色（1-2 次對話）
**交付**：登入頁、JWT 簽發、4 個角色路由保護、後台 user 種子腳本（建 4 個帳號）。

### Phase 2：資料匯入（1-2 次對話）
**交付**：`scripts/import_excel.py` — 把 `110-115年 總表.xlsx` 12 個 sheet 匯入 `records` 表。處理：
- 欄位對應（41 欄 → schema 欄位）
- 民國年轉西元年
- 資料清洗、跳過空白列
- 跑後產出匯入報告（成功/失敗、總筆數）

### Phase 3：類 Excel 主介面（2-3 次對話）
**交付**：用 Univer 做主總表頁。
- 12 個 tab 對應 12 種 category
- 41 欄完整顯示
- 儲存格直接編輯 → 立即 PATCH 後端
- 排序 / 篩選 / 凍結欄
- 4 個角色看到不同的編輯權限

### Phase 4：續約偵測 + 狀態燈（1-2 次對話）
**交付**：
- 後端 APScheduler 每日 06:00 跑 `compute_renewal_status()`
- 規則：同 `(holder_name, tax_id)`，找有沒有 `period_end > self.period_end` → 已續約
- 「續約資料」分類頁：依月份 1-12 月分頁，每月顯示「未續約名單」「續約名單」兩張清單
- 狀態燈：紅 / 綠（在 Univer 的儲存格塗色）

### Phase 5：核發 Word 套印（2-3 次對話）
**交付**：
- 模板註冊機制：把 27+ 個 `.docx` 上傳到 `templates/`、設定對應 category
- 核發頁面：輸入發票號碼 → PATCH 後端 → `issuance_status='綠'`
- 「產生」按鈕：依 category 找模板、用 docxtpl 填入欄位 → 產出證書.docx + 信封.docx
- 兩個檔案存 `generated_files/`，前端提供下載

### Phase 6：寄信檔（承辦端）（1-2 次對話）
**交付**：
- 從「未續約名單」「續約名單」勾選列 → 點「產生寄信檔」
- 用 `承辦-續約函模板.docx` 填入：(寄證地址)收件人、地址、郵區、確認(掛號/平信)
- 同名 `applicant_name`（代辦人）自動合併批次
- 產出 通知函.docx + 信封.docx

### Phase 7：搜尋 + 報表（1 次對話）
**交付**：
- 全欄位搜尋 bar（輸入文字 → 跨 12 個 category 全表撈）
- 「代辦人 → 代辦公司清單」交叉視圖
- 簡易報表頁（月/年產證書數、收入金額、續約率）

### Phase 8：部署 + SOP（1-2 次對話）
**交付**：
- DigitalOcean 一次性建置腳本（cloud-init）
- Caddy 自動 HTTPS 設定範本（你只需 DNS 指過去）
- 每日 PG dump 排程腳本 + S3 / Wasabi 異地備份範例
- 維運手冊：怎麼重啟、怎麼還原備份、怎麼加使用者、怎麼換模板

## 五、預期成本

### 一次性
- 建置：0（我寫，但你要花時間試）
- 域名：NT$300-1,000/年（如 `.tw` 域名）

### 每月
- DigitalOcean Droplet（2GB RAM / 50GB SSD）：~NT$370/月
- DigitalOcean Managed Postgres：~NT$460/月
- 備份儲存（Wasabi 1TB）：~NT$200/月
- **合計：~NT$1,030/月（≈ NT$12,400/年）**

### 跟 Option A（M365）相比
- A：~NT$2,500/月（4 人 Power Apps 授權）
- B：~NT$1,030/月（自架）
- 差距：B 月省 NT$1,500，但要自己背 2GB Droplet 的維運責任。

## 六、風險與緩解

| 風險 | 影響 | 緩解 |
|---|---|---|
| Droplet 掛掉 | 服務中斷 | 每日 dump + Wasabi 異地、預先寫好還原 SOP |
| Univer 公式不熟 | 用戶換工具不習慣 | 先把核心 CRUD 做穩，公式列保留但不強推 |
